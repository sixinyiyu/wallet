# Troubleshooting

## Common Pitfalls

### NDK or Toolchain Missing

If builds fail with NDK-related errors:
```bash
just install-ndk
just install-toolchains
```
The project requires specific NDK and Rust toolchain versions for JNI compilation.

### GitHub Packages Authentication

Gradle pulls dependencies from GitHub Packages. If dependency resolution fails, verify `local.properties` contains:
```
gpr.username=<your-github-username>
gpr.token=<your-github-pat>
```
The token needs `read:packages` scope.

### Gradle Daemon Memory

Long sessions can exhaust the Gradle daemon heap. If builds become slow or OOM:
```bash
./gradlew --stop
./gradlew clean
```

### Generated Model Drift

Never edit files generated by TypeShare or UniFFI. If Android models look wrong after a Core change:
```bash
cd .. && just generate
```
Then rebuild and re-run tests.

Generated Android artifacts typically live in:
- `gemcore/src/main/kotlin/com/wallet/core/primitives/generated/` for TypeShare-generated primitives
- Kotlin files with a `// Generated by typeshare` header
- `../core/gemstone/android/gemstone/src/main/java/uniffi/` for UniFFI-generated bindings used by the `:gemstone` module

If you need Android-specific behavior on top of generated types, add adapters, mappers, or extension functions in regular source files. Do not patch generated output to make builds or tests pass.

### Room Migration Tests

Database migration tests require an emulator or device. Run them separately:
```bash
just test-integration
```
Do not skip migration tests when changing Room schemas — they catch data loss bugs that unit tests cannot.

## Common Recovery Commands

### Build Failures

```bash
./gradlew clean
just bootstrap
./gradlew build
```

### Submodule Issues

If `gemcore/` or `core/` references are stale after a branch switch:
```bash
cd .. && just setup-git
```
Then rebuild: `./gradlew clean && just bootstrap`.
