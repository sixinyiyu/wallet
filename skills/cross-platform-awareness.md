# Cross-Platform Awareness

Both apps depend on the same Rust core and should stay aligned on shared product behavior.

## Rules

1. After any `core/` change, regenerate and verify both apps before claiming the task is done:
   ```bash
   just generate
   just ios build
   just android build
   ```
   A passing Rust build does not mean the apps still compile.
2. Implement parity or surface the gap. If a user-facing change should exist on both platforms, either implement both sides in the same task, or state the parity gap explicitly so it is not silently dropped.
3. Never edit generated files directly. Files marked `Generated by typeshare` come from `core/` — change the source, regenerate, and commit the regenerated output. Generated file locations:
   - **iOS**: `Packages/Primitives/Sources/Generated/` (TypeShare models), `GemstoneFFI.xcframework` (UniFFI binary)
   - **Android**: `gemcore/src/main/kotlin/com/wallet/core/primitives/generated/` (TypeShare models), `core/gemstone/android/gemstone/src/main/java/uniffi/` (UniFFI bindings)
   - **Pattern**: any file with a `// Generated by typeshare` header or under `uniffi/gemstone/`
4. Run `just localize` when changing shared localization inputs. Hardcoded user-visible strings on either platform are a bug.
